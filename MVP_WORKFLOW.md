# AI Stylist Agent - MVP 분석 및 워크플로우

이 문서는 현재 "AI Stylist Agent" MVP에 대한 상세 분석입니다. 기존 코드베이스를 바탕으로 앱의 기능, 의도된 사용자 경험("분위기"), 그리고 구체적인 사용자 워크플로우를 설명합니다.

## 1. 앱 컨셉 및 "분위기" (Vibe)
**테마:** "프리미엄 퍼스널 스타일리스트"
-   **비주얼:** 다크 모드 (`AppTheme.darkTheme`), 글래스모피즘 효과(반투명 카드), 부드러운 진입 애니메이션 (텍스트 페이드/슬라이드).
-   **톤앤매너:** 전문적이며, 격려하는 어조 ("오늘 비를 뚫고 나갈 준비 되셨나요?"), 그리고 스마트함.
-   **목표:** *상황(날씨/일정)*과 *옷장 데이터*를 결합하여 옷 입기 결정 피로를 해결.

## 2. 핵심 기능 분석

MVP는 4가지 주요 기능 영역으로 구성됩니다:

### A. 지능형 홈 대시보드 (`HomeScreen`)
*   **상황 인식:** 현재 날씨("18°C, 비")와 일정("이벤트 3개")을 표시하여 추천의 근거를 제시합니다. *(참고: 현재는 데모용 하드코딩 데이터 사용)*.
*   **"오늘의 픽" (Today's Pick):** 홈 화면의 중심. AI가 선택한 최적의 코디를 큰 카드로 보여줍니다.
    *   **인터랙티브:** 마음에 들지 않으면 "새로고침(Refresh)"하여 다른 추천을 받을 수 있습니다.
    *   **상세 정보:** 카드를 탭하면 `OutfitDetailScreen`으로 이동하여 *선택 이유*("매칭 점수", "추천 사유")를 확인할 수 있습니다.
*   **AI 채팅:** 특정 요청(예: "오늘 데이트 있어")을 위한 하단 채팅 인터페이스.
    *   **상태:** 모의 응답(Mocked response)이 포함된 인터랙티브 UI.

### B. 코디 상세 보기 (`OutfitDetailScreen`)
*   **탐색:** 추천받은 코디를 자세히 살펴봅니다.
*   **구성 요소:** 룩을 개별 아이템(상의, 하의)으로 분해하여 보여줍니다.
*   **투명성:** "매칭 점수"(예: 98%)와 텍스트 설명을 통해 신뢰를 구축합니다.
*   **액션:** "캘린더에 저장" 버튼 (현재는 시각적 요소).

### C. 디지털 옷장 (`ProfileScreen` & `WardrobeNewScreen`)
*   **아이템 추가:** 옷을 디지털화하는 전용 플로우.
    *   **메커니즘:** 사진 선택 -> 앱이 "AI 분석" 시뮬레이션(진행바) -> 속성 자동 태깅(카테고리, 색상).
*   **탐색:** "상의", "하의" 등으로 필터링 가능한 옷장 그리드 뷰.
*   **통계:** 신체 프로필 데이터 표시 (키: 182cm, 몸무게: 75kg).

### D. 아이템 상세 보기 (`ItemDetailScreen`)
*   **메타데이터:** 사용 통계 ("12회 착용", "마지막 착용 2일 전") 표시.
*   **속성:** 카테고리, 색상, 브랜드 정보 표시.

## 3. 사용자 워크플로우

구현된 화면을 바탕으로 사용자가 수행하는 주요 워크플로우는 다음과 같습니다:

### 워크플로우 1: "외출 준비" (아침 루틴)
가장 핵심적인 사용 사례입니다.
1.  **앱 실행:** 사용자 맞춤 인사말과 상황 정보(날씨/이벤트) 확인.
2.  **추천 확인:** 시스템이 "오늘의 픽"(예: 비 오는 날 코디)을 제시.
3.  **결정:**
    *   **마음에 들면:** 카드 탭 -> 개별 아이템 확인 -> 착용.
    *   **마음에 안 들면:** 홈 화면의 **새로고침 아이콘 (🔄)** 탭 -> 시스템이 새로운 옵션 제시.

### 워크플로우 2: "옷장 채우기" (온보딩/관리)
1.  **시작:** 홈 화면이나 옷장 탭의 **(+) FAB** 버튼 탭.
2.  **촬영/선택:** 갤러리에서 사진 선택 또는 카메라 촬영.
3.  **처리:**
    *   "AI 분석 중..." 메시지와 진행바 표시.
    *   체크마크 ✅ (분석 완료) 대기.
4.  **확인:** 자동 감지된 태그 확인 (예: "검정 티셔츠").
5.  **저장:** "완료(Finish)" 탭 -> `ProfileScreen`의 옷장 그리드에 아이템 추가됨.

### 워크플로우 3: "특별한 이벤트" (채팅)
1.  **접근:** 홈 화면을 아래로 스크롤하여 채팅 섹션으로 이동.
2.  **요청:** 요청 사항 입력 (예: "격식 있는 저녁 식사 코디해줘").
3.  **수신:** AI가 (지연 시간 시뮬레이션 후) 해당 요청에 맞춰 새로운 코디를 큐레이팅했음을 확인해주는 응답 제공.

## 4. 기술적 상태 요약
| 기능 | 구현 상태 | 비고 |
| :--- | :--- | :--- |
| **UI/UX** | 🟢 High Fidelity | 애니메이션, 글래스모피즘, 반응형 레이아웃 적용됨. |
| **내비게이션** | 🟢 완료 | 하단 탭, 스택 내비게이션, 제스처 작동. |
| **데이터 (날씨)** | 🟡 모의(Mock) | `HomeScreen` 위젯에 하드코딩됨. |
| **데이터 (코디)** | 🟡 모의/Provider | `ApiService`에서 JSON 가져옴 (모의 데이터). |
| **카메라/갤러리** | 🟢 작동 | `image_picker` 플러그인 사용. |
| **AI 분석** | 🔴 시뮬레이션 | 진행바는 타이머임; 속성은 랜덤/모의 데이터. |
